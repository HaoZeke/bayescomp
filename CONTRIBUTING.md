# Contributing to bayescomp

Thank you for considering contributing to bayescomp.

## How to contribute

1. Open an issue at <https://github.com/HaoZeke/bayescomp/issues> to discuss
   your proposed change before starting work.
2. Fork the repository and create a feature branch from `main`.
3. Follow the existing code style and ensure `R CMD check` passes with no
   errors or warnings.
4. Add or update tests in `tests/testthat/` for any new or changed
   functionality.
5. Submit a pull request against `main`.

## Reporting bugs

Please open an issue with a minimal reproducible example.

## Maintenance

This package is actively maintained by Rohit Goswami (rgoswami@ieee.org).
The maintainer commits to responding to issues and pull requests for at
least two years from the date of initial rOpenSci acceptance, and to
finding a new maintainer or archiving the package if that commitment
cannot be met.
